# Client side validation

Even the simplest of forms will usually benefit from some sort of client validation.
This example builds onto the previous one and adds a simple client side validator to both fields.

Also, as our form now handles validations, so we should use the supplied "submit" method and pass in our actual method
that sends the information to server as the "onSubmitForm" argument. This makes the server protected, "onSubmitForm" won't get
called unless all fields passed validation.

```js
// SignIn.tsx

function SignIn() {
  const { form, update, submit } = useForm({
    initial: {
      username: "",
      password: "",
    },
    clientValidators: {
      username: ({ username }) =>
        username.length == 0
          ? new Error("Must contain at least 1 char.")
          : undefined,
      password: ({ password }) =>
        password.length == 0
          ? new Error("Must contain at least 1 char.")
          : undefined,
    },
    onSubmitForm: ({ username, password }) => {
      /// This scope guarantees all fields passed validations.
      /// Send form.username + form.password to server.
    },
  });

  return (
    <>
      <input
        value={form.username}
        onChange={(e) => update("username", e.target.value)}
      />
      <input
        value={form.password}
        onChange={(e) => update("password", e.target.value)}
      />
      <button onClick={() => submit()}>Submit</button>
    </>
  );
}
```
